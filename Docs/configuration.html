Configuration

Z offers various ways to customize your editing experience. This guide covers the available configuration options and how to apply them.

Settings Files

Z uses the following JSON configuration files:

File Purpose Location
settings.json User preferences including language, theme, and editor behavior %AppData%\Local\Z\ (Windows)
~/.local/share/Z/ (Linux)
~/Library/Application Support/Z/ (macOS)
keybindings.json Custom keyboard shortcuts Same as settings.json
analytics-settings.json Anonymous usage data collection preferences Same as settings.json
Note: The configuration files are automatically created with default values when they don't exist.

Editor Settings

The settings.json file allows you to customize various aspects of Z. Here are the available options:

Interface Preferences

{
  "ui": {
    "theme": "light", // "light" or "dark"
    "fontSize": 14,
    "fontFamily": "SF Mono, Monaco, Consolas, monospace",
    "lineHeight": 1.5,
    "showLineNumbers": true,
    "showIndentGuides": true,
    "showWhitespace": false,
    "minimap": {
      "enabled": true,
      "size": "proportional" // "proportional" or "fixed"
    }
  }
}

Editor Behavior

{
  "editor": {
    "tabSize": 2,
    "insertSpaces": true,
    "wordWrap": "off", // "off", "on", "wordWrapColumn", "bounded"
    "wordWrapColumn": 80,
    "autoIndent": "full", // "none", "keep", "full"
    "formatOnSave": false,
    "formatOnPaste": false,
    "trimTrailingWhitespace": true,
    "insertFinalNewline": true
  }
}

Language Preferences

{
  "language": "en", // Language code (e.g., "en", "ru")
  "languageSpecific": {
    "javascript": {
      "tabSize": 2,
      "formatOnSave": true
    },
    "python": {
      "tabSize": 4,
      "insertSpaces": true
    }
  }
}

File Associations

{
  "files": {
    "associations": {
      "*.jsx": "javascriptreact",
      "*.ts": "typescript",
      "*.mdx": "markdown"
    },
    "exclude": {
      "**/.git": true,
      "**/.DS_Store": true,
      "**/node_modules": true,
      "**/.vscode": true
    },
    "encoding": "utf8",
    "eol": "\n" // "\n" for LF, "\r\n" for CRLF
  }
}

Keyboard Shortcuts

You can customize keyboard shortcuts in the keybindings.json file:

[
  {
    "key": "ctrl+shift+f",
    "command": "workbench.action.findInFiles"
  },
  {
    "key": "ctrl+`",
    "command": "workbench.action.terminal.toggleTerminal"
  },
  {
    "key": "alt+z",
    "command": "editor.action.toggleWordWrap"
  }
]

Each keybinding has the following properties:

  • key: The key combination (e.g., ctrl+shift+f)
  • command: The command ID to execute
  • when (optional): Context when the keybinding is active

Key Modifiers

Modifier Windows/Linux macOS
ctrl Control key Control key
shift Shift key Shift key
alt Alt key Option key
cmd Windows key Command key

Analytics Settings

Z can collect anonymous usage data to help improve the editor. You can control this through the analytics-settings.json file:

{
  "basicAnalytics": true, // Basic usage data (version, OS, features used)
  "systemAnalytics": false // System information (screen resolution, architecture)
}
Privacy Note: Z never collects personal information or file contents.

Z-Rosetta Language Settings

As described in the Z-Rosetta documentation, language preferences are stored in the main settings.json file:

{
  "language": "en" // Language code (e.g., "en", "ru")
}

Project-Specific Settings

Z also supports project-specific settings through a .z/settings.json file in the project root. Settings in this file override the global settings, but only for that specific project.

Example project settings:

{
  "editor": {
    "tabSize": 4,
    "insertSpaces": true,
    "formatOnSave": true
  },
  "files": {
    "exclude": {
      "**/build": true,
      "**/dist": true
    }
  }
}

Configuration UI

While you can edit the JSON files directly, Z also provides a settings UI:

  1. Open Z and click on the settings gear icon in the status bar
  2. Navigate through the categories in the settings panel
  3. Changes are saved automatically
Note: Some advanced settings are only available through the JSON files.

Resetting to Defaults

To reset Z to default settings:

  1. Close Z
  2. Delete or rename the configuration files
  3. Restart Z (new default configuration files will be created)
